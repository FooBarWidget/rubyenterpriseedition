Tested platforms
----------------
Ubuntu Linux 8.04 (x86)
	Supported                : Yes
	tcmalloc works           : Yes
	REE is linked to tcmalloc: Yes
	uname -a                 : Linux localhost 2.6.24-21-generic #1 SMP Tue Oct 21 23:43:45 UTC 2008 i686 GNU/Linux
	
Ubuntu Linux 8.10 (x86_64)
	Supported                : Yes
	tcmalloc works           : No
	REE is linked to tcmalloc: No
	uname -a                 : Linux ubuntu 2.6.27-7-server #1 SMP Fri Oct 24 07:20:47 UTC 2008 x86_64 GNU/Linux
	
	tcmalloc can't generate stack traces without libunwind, but libunwind
	bails out with a compilation error. So REE works on this platform, but
	without tcmalloc.

MacOS X 10.5.5 server (x86)
	Supported                : Yes
	tcmalloc works           : Yes
	REE is linked to tcmalloc: No
	uname -a                 : Darwin localhost 9.5.0 Darwin Kernel Version 9.5.0: Wed Sep  3 11:29:43 PDT 2008; root:xnu-1228.7.58~1/RELEASE_I386 i386
	
	Because of OS X's two-level namespace scheme for symbols, Ruby ends up
	using tcmalloc to free memory that's allocated by the system's allocator.
	tcmalloc cannot made to work without further modifications to Ruby, and
	so is disabled for now.
